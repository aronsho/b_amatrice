# General Behavior

Load this file when the task needs repo-specific working style beyond the short root `AGENTS.md`.

## Working Style

- Prefer small, clear changes over broad rewrites unless the current structure is the real problem.
- If the current approach is structurally wrong, simplify it instead of adding more layout or control logic around it.
- When the user asks why something does not work, explain the actual cause, not just the symptom.
- If a repeated problem is caused by shared code, fix the shared helper first.
- Avoid wasting tokens on unrelated context. Read only the project instruction file that matches the task.

## Collaboration Style

- Be direct and practical.
- Do the work instead of giving long speculative plans.
- When visual output is involved, optimize for legibility and consistency, not cleverness.
- If data is missing, say so plainly in the output instead of implying that a blank panel is meaningful.
